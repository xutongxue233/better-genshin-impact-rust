# Rust Migration Plan

This repository is moving to a full Rust implementation. The original C#/.NET implementation remains only as a temporary behavior reference. It must be removed when parity is proven so the repository has a single Rust code path.

## Workspace

- `crates/bgi-core`: platform-neutral core models and helpers.
- `crates/bgi-script`: script manifests, scheduler group models, project loading/module search models, settings schemas, key/mouse macro models, host API descriptors, execution plans, schedule models, policy gates, and cancellation semantics.
- `crates/bgi-script-engine`: Rust JavaScript execution boundary backed by Boa, including classic-script source execution, `settings` injection, console/log capture, host-call recording, and prepared execution outcome models.
- `crates/bgi-input`: SendInput-compatible input event model, PostMessage background input model, Genshin action key-binding translation, and Windows dispatch boundary.
- `crates/bgi-hotkey`: legacy hotkey parser and Windows RegisterHotKey boundary.
- `crates/bgi-capture`: game capture mode model, frame model, and Windows BitBlt capture boundary.
- `crates/bgi-vision`: recognition object model, OCR result model, BvPage/BvLocator/BvImage host type models, ImageRegion/GameCaptureRegion-style region models and executable BGR24 image regions, packed BGR24 image decoding/PNG writing/crop/nearest-resize, pure Rust BGR24 template matching, BGR/RGB/Gray/HSV color-range matching, template-match semantics, and ONNX model registry.
- `crates/bgi-task`: task runtime state, dispatcher trigger selection, unified task catalog, independent task descriptors, task parameter models, independent task execution boundary, and progress model.
- `crates/bgi-cli`: migration CLI for checking config, assets, triggers, and route JSON.
- `apps/desktop`: Tauri 2 desktop shell with a Svelte + TypeScript operational UI.
- `BetterGenshinImpact`, `Fischless.*`, and `Test`: legacy C# sources retained temporarily. They are not part of the final Rust deliverable.

## UI Decision

Primary UI stack: Tauri 2 + Svelte + TypeScript.

Reasoning:

- Rust remains the native application boundary for capture, input, hotkeys, OCR, task execution, filesystem, tray, and update logic.
- BetterGI needs dense settings pages, script editing, map/pathing tools, logs, tables, and WebView-style content. Svelte is a better fit for those interaction-heavy views than rebuilding every control in a pure Rust widget toolkit.
- The existing application already relies on WebView-style workflows, so Tauri minimizes UI migration risk while still removing the .NET/WPF runtime.

Pure Rust UI fallback: Slint. Use it only if a no-webview UI becomes a hard requirement.

UI shape:

- First screen is the control console, not a landing page.
- Left navigation follows the legacy WPF structure: Startup, Realtime Triggers, Independent Tasks, One Dragon, Automation, Control Assist, Hotkeys, Notifications, Settings.
- Work surfaces are dense and task-oriented: tables, split panels, command bars, status badges, logs, pathing editors, and settings forms.
- No marketing hero, no decorative card-heavy composition, no one-note purple/dark palette.
